rust by push triangle

値が破棄されるのはどんなときかをみる。

dropが呼ばれると、中を見せつつびっくりする構造体を次のように定義する。

struct A(usize);

impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}

fn main() {
  let a: A = A(1);
}

A(n)という値が破棄されると n!という文字列が出力される。 これで値が破棄されたときがわかるようになる。

場所の終了の基本

場所が終了すると値が破棄される

struct A(usize);
impl Drop for A {
 fn drop(&mut self) {
   print!("{}!", self.0)
 }
}
fn main() {
  let a: A = A(1);
}

a という変数（場所）が main 関数の終わりに終了し、 a の持っていた値が破棄された、つまり Drop の実装が呼ばれる。

スコープ終わりに場所が終了する

struct A(usize);
impl Drop for A {
 fn drop(&mut self) {
   print!("{}!", self.0)
 }
}
fn main() {
  let a: A = A(1);
  {
    let a2: A = A(2);
  }
}

2!1!

ざっくりいうと、 { と } で囲まれた部分について、その中で宣言された変数は } のところで終了する。 詳しくは次でやる。

値の転送元と破棄

moveで元の場所にある値の破棄は起こらない例

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let x = A(1);
  let y = x;
}

1!

yが有効でなくなった分しか、printがされてない。 xという場所は3行目で値をyに移した...もっていた値がなくなったので、x は破棄するべき値がないまま終了する。

copyの場合には？

copy の場合には、値は元の場所にも残っている（として扱われる）ので、破棄する必要があるはず。 ただし、値の破棄に際して特別な処理を行うこと（= Drop）を行うことができない。 コードの意味や処理上では、値の破棄は行われる（なにかしら drop関数が呼ばれる）が、 それは何もせずただ 値が使えなくなる ものとして処理される。 drop関数の実装などを参照すること。

値の転送先と破棄

代入先に値がすでに存在した場合、その値は破棄される。

再代入で転送先の値が破棄される例

mutable な変数 x への再代入（ちゃんとした言葉だっけ？）をやってみる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let mut x = A(1);
  x = A(2);
}

1!2!

xがもともと持っていた値 A(1)が破棄されてA(2)が入った。 その後に xが有効でなくなったので A(2)も破棄された。

shadowing の場合

いわゆる shadowing の場合は、新たに変数を作るため、上とは異なる結果になる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let mut x = A(1);
  let x = A(2);
}

2!1!

場所の終了 => 値の破棄

変数のスコープの終了時には、中にあった値は破棄される。 値がないときは当然値の破棄が起こらない。

変数の終了時の例

例えば、プログラムの終了時（mainが最後に到達したとき）

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let x = A(1);
  print!("hello");
}

hello1!

他には、スコープを定義すると、そのスコープの終わりには破棄される。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  {
    let x = A(1);
  }
  print!("hello");
}

1!hello

ややこしい一時変数とか placeholder とか。

一時変数の場合の例

次のプログラムに書かれた A(1) も所有者（場所）はいるはずだが、すぐに終了する。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  A(1);
  print!("hello");
}

1!hello

hello1!ではないことに注意

値を持っていなかった場合

値のない場所の終了時には何も起きない。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let x: A;
}

なんの出力もされない。

Copy と Drop は共存不可

copy な値は、 drop 時に特殊な処理を行うことができない。 次はコンパイル不可。

コンパイル通らない例

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
      print!("{}!", self.0)
  }
}

impl Clone for A {
  fn clone(&self) -> Self {
    A(self.0)
  }
}

impl Copy for A {}

fn main() {
  let x = A(1);
  let y = x;
}

error[E0184]: the trait Copy cannot be implemented for this type; the type has a destructor

destructor は Drop トレイトの drop関数のこと
