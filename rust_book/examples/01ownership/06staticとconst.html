static と const

Drop のタイミングがわかる構造体を使う。

#![allow(unused)]
fn main() {
struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
}

static の使い方

staticに値を定義して値を用いる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
static S: A = A(1);
fn main() {
}

何も出力されない、これは S という定数が Drop されないから。 static な場所は、プログラム中で終了しない。 ちなみに、 let x = S;に変えてもってこようとするとコンパイルエラーになる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
static S: A = A(1);
fn main() {
    let x = S;
}

error[E0507]: cannot move out of static item S

それはそう。 Sという場所は常に値をもっていなければいけないだろう。 でも、 let _ = S;にするとコンパイルエラーにならないし、何も出力されない。 let _ = ... は値を移していない。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
static S: A = A(1);
fn main() {
    let _ = S;
}

const の使い方

const の場合も似たような形で宣言できるが、効果は異なる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
const S: A = A(1);
fn main(){
}

この場合は何も出力されない。 これは定数 S はmain関数中に登場しなかったから でも let _ = S;にすると出力される。 この挙動は static と異なる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
const S: A = A(1);
fn main() {
    let _ = S;
}

1!

S が Drop されている。 しかも、 let x = S;に変えてもコンパイルエラーにならない、どころか S を move するように見える式を書いても全然大丈夫。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
const S: A = A(1);
fn main() {
    let x = S;
    let y = S;
}

1!1!

つまり、 constは場所ではなく値を作り出す式のことと思ってよい。

temporary の promotion

temporary を 'static として扱うことができる。

fn main() {
  let x: &'static usize = &1;
}

ただし、右辺が const epxression じゃないとだめ。

fn main() {
  let x: &'static String = &String::new();
}

String::new() は const だが、だめらしい。 ちゃんと const にする。

fn main() {
  const S: String = String::new();
  let x: &'static String = &S;
}

これもだめ。一応なぜだめかとかはいろいろ決まってる。 ref これによると、 Drop がついているものはだめ。 String はヒープ管理文字列だから、Drop が確かに特殊なので、だめ。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let x: &'static A = &A(1);
}

確かに Drop ついているとだめだ。

generalized な型の場合

これがコンパイルが通る。

#![allow(unused)]
fn main() {
fn f<T>() -> &'static Option<T> {
  &None
}
}

呼び出す側が気を付けることになりそう。

fn f<T>() -> &'static Option<T> {
 &None
}
fn main() {
  let x = f::<usize>();
}
fn f<T>() -> &'static Option<T> {
 &None
}
fn main() {
  let x = f::<String>();
}
