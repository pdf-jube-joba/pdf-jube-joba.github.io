rust by push triangle

場所が値を持っているとき、「場所が有効である」ということにして、 場所が有効な期間と場所が存在する期間の差について考える。

move したからといって場所が終了するわけではない

move な値について、ある場所からある場所へ値を移したとしても、もとの場所が即座に終了するわけではなく、値を失っただけ。

値を移した後の再代入の例

struct A(usize);
fn main() {
    let mut x = A(1);
    let y = x; // `x`は値を失った
    x = A(2); // `x` は再び値を所有している。
}

xという場所から値を移したがその後にxという場所を使っている。 xという場所はいったん値を失ったものの、その時点で終了はしていない。

使う際にはまた値が必要になる例

xに再代入を行う前にxという場所を使うことはできない。

struct A(usize);
fn main() {
    let mut x = A(1);
    let y = x;
    print!("{}", x.0);
    x = A(2);
}

error[E0382]: borrow of moved value: x

変数のスコープとブロックについて詳しく

変数は宣言されてから終了するまでのコード上の領域を持ち、これをスコープという。 Dropされた（値が破棄された）タイミングがわかる構造体を使う。

スコープ内での順番

宣言順と終了順の例

関数が終わるときには、スコープが終わる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
  let x = A(1);
  let y = A(2);
}

2!1!

値の破棄の順序は、宣言された順番と逆 ということは、次のようなコードと対応しているはず

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    {
        let x = A(1);
        {
          let y = A(2);
        }
    }
}

これがスコープの基本だと思う。

初期化を遅らせる例

変数の宣言の順番が重要で、初期化の順番は関係ない。 次の2つのプログラムは、 y という場所の終了後に x という場所の終了が起きる。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    let x: A;
    let y: A;
    y = A(1);
    x = A(2);
}

1!2!

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    let x: A;
    let y: A;
    x = A(2);
    y = A(1);
}

1!2!

スコープと場所の宣言

ブロックでスコープを区切る例

適当に {と}で囲むと変数は有効な領域がそこに限定される

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    {
        let x = A(1);
    }
    {
        let y = A(2);
    }
}

1!2!

xが終了してからyが終了するのでこの順番で出力される。

{...} の外で宣言して中で初期化する例

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    let x: A;
    {
        let y = A(2);
    }
    {
        x = A(1);
    }
    {
        let z = A(3);
    }
}

2!3!1!

  1. x が宣言される。
    1. y が宣言される。
    2. y が A(2) で初期化される。
    3. y が終了する => A(2) が破棄される。
    1. x が A(1) で初期化される
    1. z が宣言される。
    2. z が A(3) で初期化される。
    3. z が終了する => A(3) が破棄される。
  5. x が終了する => A(1) が破棄される。

関数に渡すとき

関数のスコープの例

関数に渡したらその関数が終わったタイミングで破棄される。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn f(a: A) {}
fn main() {
    let x = A(1);
    let y = A(2);
    f(x);
    f(y);
}

1!2!

f(x)と呼ばれると、

  • 場所x から引数aへのmoveが起きる。
  • 関数fが始まる。
  • 関数fが終わる。
  • 引数fは場所 a の中に値 A(1) を持っているので値の破棄が起きる。
  • 1!が出力される。
  • その後に f(y)で同じことが起きる。

一時変数

束縛されない場合は即座に破棄される例

よくわからないけど、変数への束縛がない場合には、その場で値が破棄される。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    A(1);
    A(2);
}

1!2!

let _ = の例

_ は let _ =と同じ扱い。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    let _ = A(1);
    let _ = A(2);
}

1!2!

unused variable の例

変数に _ をつけることで 使われない変数を表し、 コンパイラの警告を黙らせることができる。 変数が確保されている ことに注意する。

struct A(usize);
impl Drop for A {
  fn drop(&mut self) {
    print!("{}!", self.0)
  }
}
fn main() {
    let _a1 = A(1);
    let _a2 = A(2);
}
