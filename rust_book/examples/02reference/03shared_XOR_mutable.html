rust by push triangle

参照をつくるときには、どの時点でも次のうちのどちらか一方のみが成り立っていないといけない。

  • 不変参照が複数ある。
  • 可変参照が1つある。

また、可変参照は、書き込みを行う権限をまるごと代理で任せてしまうので、 もとの場所は書き込みをする主体にはなれない。 ここの時点という書き方はあいまいだが、コード上も実行時も両方の意味で。

不変参照の例

不変参照をたくさん作る 不変な参照（&のやつ）はいくつ作ってもいい。

fn main() {
  let x: usize = 1;
  let y = &x;
  let z = &x;
  print!("{}", *y);
  print!("{}", *z);
}

不変参照を copy でつくる。

ちなみに、不変な参照は Copy である。 なので、次のように書いてもよい。

fn main() {
  let x: usize = 1;
  let y = &x;
  let z = y; // &usize は move ではなく copy ... 場所 `y` に入っている値 `&x` が場所 `z` に copy される。 
  print!("{}", *y); // y が使える。
  print!("{}", *z);
}

可変参照はたくさんつくれない。

たくさんつくろうとするとコンパイルできない。

fn main() {
  let mut x: usize = 1;
  let y = &mut x;
  let z = &mut x;
  print!("{}", *y);
  print!("{}", *z);
}

error[E0499]: cannot borrow x as mutable more than once at a time

ちなみに、 print!(...)を両方消すとコンパイルが通る。

fn main() {
  let mut x: usize = 1;
  let y = &mut x;
  let z = &mut x;
}

rustが最大限好意的に解釈してくれるから大丈夫だと思われる。 下のように解釈されている？

fn main() {
  let mut x: usize = 1;
  {
    let y = &mut x;
  } // `y` 終了
  // x の不変参照は1つもない状態になった。
  {
    let z = &mut x;
  }
}

あるいは、 y に値が入ってない状況になったか。

可変参照は copy でない

可変参照はcopyでないので、moveされてしまい値が失われる。 次はコンパイルが通らない。

fn main() {
  let mut x: usize = 1;
  let y = &mut x;
  let z = y; // `y` から `z` への move がおきる
  print!("{}", *y);
}

error[E0382]: borrow of moved value: y

不変参照と可変参照は同時に作れない

fn main() {
  let mut x: usize = 1;
  let y = &mut x;
  let z = &x;
  print!("{}", *y);
  print!("{}", *z);
}

error[E0502]: cannot borrow x as immutable because it is also borrowed as mutable

参照があるときは値の変更はできない。

fn main() {
  let mut x = 1;
  let z = &x;
  x = 2;
  print!("{}", *z);
}

error[E0506]: cannot assign to x because it is borrowed

fn main() {
  let mut x = 1;
  let z = &mut x;
  x = 2;
  print!("{}", *z);
}

error[E0506]: cannot assign to x because it is borrowed

参照という値が失われる例

fn main() {
  let mut x: usize = 1;
  let mut y: &mut usize = &mut x;
  {
    let z = y; // y の持っている`x`への可変参照は move された
    // `y` には値が入っていない
  } // `x`への可変参照は失われた。
  x = 2; // `x` を変えてもよい。
}

yは moved value の扱いになり、 x は borrow されている扱いになる。

fn main() {
  let mut x: usize = 1;
  let mut y: &mut usize = &mut x;
  {
    let z = y;
  }
  x = 2; 
  *y;
}

error[E0506]: cannot assign to x because it is borrowed
error[E0382]: use of moved value: y

可変参照を不変参照にしたら、読み書きしていいのか

fn main() {
  let mut x: usize = 1;
  let y: &mut usize = &mut x;
  x = 2;
}

これぐらいなら追跡できるので、 y に入っていた値を適当に無効にしたと考えて、 &mut x という書き込み権が消えたことはわかるわけだ。

fn main() {
  let mut x: usize = 1;
  let mut y: &usize;
  {
    y = &mut x;
  }
  x = 2;
}
